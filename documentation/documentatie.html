1. Introducere

Child monitor este o aplicatie web ce le ofera userilor posibilitatea de a monitoriza locatia in timp real a unor entitati.

1.1 Scop

Scopul aplicatiei Child Monitor este de a oferi mai multor useri posibilitatea de a isi introduce datele si a isi urmari unele entitati.

1.2 Conventiile proiectului

La realizarea aplicatiei s-au respectat conventiile de utilizare HTML si CSS, partea de back-end a proiectului a fost realizata dupa modelul Model - View - Controller iar codul va fi verificat cu ajutorul validatoarelor online Validator W3 Org si CSS Lint. Se respecta o paleta de culori adecvata publicului tinta. Niciun framework nu a fost folosit in procesul de realizare al acestei aplicatii.

2. Descriere si structura

2.1. Descriere

Aplicatia foloseste mai multe baze de date pentru a stoca informatii in legatura cu datele utilizatorului si entitatile pe care acesta le urmareste.

2.2. Functionalitati

Principalele functionalitati ale aplicatiei sunt:

  • pagina introductiva Home, de unde se poate naviga direct catre toate paginile odata ce utilizatorul este logat
  • pagina About, unde utilizatorul afla mai multe informatii despre aplicatie
  • pagina Contact unde utilizatorii pot afla mai multe despre developerii aplicatiei
  • pagina Login/Register unde utilizatorii deja inregistrati se pot autentifica, iar utilizatorii noi isi pot crea un cont nou.
  • pagina Configuration, la care au acces doar utilizatorii care s-au logat pe pagina, in care utilizatorii isi pot configura mai multe setari si in care utilizatorii pot introduce entitati noi
  • pagina Monitor, la care au acces doar utilizatorii care s-au logat pe pagina, in care se pot urmari entitatile adaugate de utilizatori

2.3 Environment de utilizare

Aplicatia va rula in browser pe orice device care poate avea acces la un browser si internet si va folosi mai multe baze de date pentru managerierea entitatilor si userilor. Este necesar un browser mai nou decat Internet Explorer si care suporta HTML5 si CSS3 pentru o buna functionare a website-ului.

2.4 Limbaje

  • Pentru partea de frontend a aplicatiei se va utiliza HTML5, CSS3, Javascript.
  • Pentru partea de backend a aplicatiei se va utiliza PHP.
  • Pentru baza de date se utilizeaza mySql.

    • 3. Interfata

    3.1 Interfete user

    NavBarul pentru interfata user va fi folosit pentru majoritatea paginilor.

    3.2 Interfete hardware

    Child Monitor este disponibil pe device-urile ce suporta utilizarea orcaruia dintre urmatoarele browsere:

    • Google Chrome (ultima versiune) si derivatele sale
    • Microsoft Edge (ultima versiune)
    • Opera/OperaGX
    • Mozilla Firefox si derivatele sale
    • Torch Browser
    • Brave
    • Vivaldi
    • si posibil multe altele... (Majoritatea browser-urilor care suporta HTML5 si javaScript)

    3.3 Interfete software

    Aplicatia creeaza o baza de date mySQL pentru a putea retine userii si entitatile. Conectarea la baza de date se realizeaza de catre server-ul de backend, proiectat in PHP. Pentru a asigura stocarea criptata a parolelor, s-a folosit un HashMap. Nu a fost folosit niciun framework pentru dezvoltarea acestei aplicatii.

    3.4 Interfete de comunicare

    Aplicatia utilizeaza protocoale de comunicare HTTP.

    Path-urile utilizate faciliteaza comunicarea dintre client (browser).

    Metodele HTTP utilizate sunt: POST,GET, DELETE

    4. Functionalitati

    4.1 Inregistrare ca utilizator nou

    Vizitatorii website-ului au posibilitatea de a se inregistra ca utilizatori. Pentru ca o persoana sa se poata inregistra este nevoie sa introduca mai multe informatii ce respecta un anumit format iar campul Username trebuie sa fie unic.
    Pentru a putea crea un cont nou, utilizatorul va utiliza butonul de register din pagina Register, iar dupa creearea contului va fi automat redirectionat catre sectiunea de Login.

    4.2 Autentificare cu un cont de utilizator

    Utilizatorii deja inregistrati care au trecut prin procesul de completare a formularului de inregistrare doar isi vor introduce numele de utilizator si parola in formularul de logare, server-ul va genera un session, iar apoi utilizatorii vor fi redirectionati catre landing page. Daca un utilizator va introduce un set incorect de credentiale pentru autentificare, pe pagina va aparea un mesaj care il va atentiona de acest lucru.

    4.4 Pagina de start(Landing page)

    Utilizatorii aplicatiei au acces la aceasta pagina, pe post si de pagina Home, de unde pot naviga catre formularul pentru generarea de sugestii de produse, locul de unde se pot deloga, pot accesa pagina pentru generarea statisticilor etc.

    4.7 Redirectionare pagini in caz de erori

    Deoarece exista cazuri in care utilizatorii pot accidental sa introduca un URL gresit din bara de adrese sau sa incerce sa acceseze o pagina care nu exista in aplicatie, in acest caz acestia vor fi redirectionati catre o pagina care ii va atentiona de acest lucru.
